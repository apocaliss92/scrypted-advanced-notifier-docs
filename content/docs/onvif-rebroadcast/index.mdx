<PluginTitle slug="onvif-rebroadcast" title="ONVIF Rebroadcast" />

Scrypted mixin plugin that exposes your cameras as fully compliant **ONVIF devices**. Any system that supports ONVIF discovery (NVRs, VMS, home automation) can automatically find and connect to your Scrypted cameras using their RTSP rebroadcast streams.

## Features

- **WS-Discovery** ŌĆö Cameras are automatically discoverable via ONVIF multicast (239.255.255.250:3702)
- **RTSP stream mapping** ŌĆö All Scrypted rebroadcast streams (main, sub, third) are exposed as ONVIF media profiles
- **Unique IP & MAC per camera** ŌĆö Docker macvlan proxy containers give each camera its own network identity (required for UniFi Protect)
- **Event forwarding** ŌĆö Motion and object detection events from Scrypted are forwarded as ONVIF events via PullPoint subscription
- **PTZ support** ŌĆö If the camera has PTZ capabilities in Scrypted, they are exposed via ONVIF PTZ service
- **Two-way audio** ŌĆö Intercom/audio output capabilities are advertised when available
- **WS-Security authentication** ŌĆö UsernameToken Digest and HTTP Basic auth support
- **Deterministic identity** ŌĆö Each camera gets a stable UUID, IP, MAC, and port across restarts
- **Per-camera configuration** ŌĆö Enable/disable, custom port, debug logging per camera

## Prerequisites

- **Scrypted Rebroadcast plugin** must be installed and running ŌĆö this plugin reads the RTSP rebroadcast URLs from it
- **Docker socket** mounted at `/var/run/docker.sock` in the Scrypted container (required for unique IP mode)

## Installation

1. Install **ONVIF Rebroadcast** from the Scrypted plugin store (or side-load the npm package)
2. Configure **Username** and **Password** in the plugin settings (optional, leave empty to disable auth)
3. The plugin automatically attaches to all cameras and doorbells
4. Cameras become discoverable via ONVIF within seconds

## Plugin Settings

| Setting | Description |
|---------|-------------|
| **Username** | Username for ONVIF authentication (leave empty to disable) |
| **Password** | Password for ONVIF authentication |
| **Auto-assign unique IPs** | Enable automatic proxy container creation for unique IPs/MACs |
| **IP range start** | First IP address to assign to cameras (e.g. `192.168.1.240`) |
| **Network interface** | Parent interface for the macvlan Docker network (e.g. `br0`) |
| **Subnet prefix length** | CIDR prefix for the macvlan network (e.g. `23` for /23) |
| **Gateway** | Default gateway for the macvlan network (e.g. `192.168.1.1`) |

## Per-Camera Settings

Each camera gets its own settings under the **ONVIF Rebroadcast** group:

| Setting | Description |
|---------|-------------|
| **ONVIF IP address** | Manual unique IP for this camera (leave empty when auto-assign is enabled) |
| **ONVIF port** | Port for this camera's ONVIF server (leave empty for auto-assignment) |
| **ONVIF server enabled** | Enable/disable the ONVIF server for this camera |
| **Debug events** | Enable verbose logging for motion and object detection events |

## UniFi Protect Setup

UniFi Protect identifies third-party ONVIF cameras by MAC address. When all virtual cameras share the host's MAC, the NVR sees them as a single device ŌĆö adding a second camera overwrites the first.

This plugin solves this by giving each camera its own **unique IP and MAC address** on the network via Docker macvlan proxy containers.

### Docker Socket

The plugin needs access to the Docker socket to create proxy containers. Add this path mapping to your Scrypted container:

| Container Path | Host Path |
|---|---|
| `/var/run/docker.sock` | `/var/run/docker.sock` |

### IP Range Selection

Choose IPs **outside your DHCP pool** on the same subnet as your UniFi controller. For example, if your router assigns `192.168.1.2ŌĆō200` via DHCP, use `192.168.1.240` as the start.

### How It Works

```
UniFi Protect                    Docker macvlan proxy              Scrypted container
(192.168.1.x)                    (192.168.1.240, unique MAC)       (192.168.4.40)
     |                                  |                                |
     |--- ONVIF (port 8000) ---------->|--- TCP proxy (port 18000) --->|  ONVIF server
     |--- RTSP  (port 554)  ---------->|--- TCP proxy (port 42917) -->|  RTSP rebroadcast
```

Each camera gets its own lightweight `alpine/socat` proxy container with:
- A unique IP address on your LAN
- A unique MAC address (deterministic, based on device ID)
- TCP proxies for both ONVIF (port 8000) and RTSP (port 554+) traffic

Proxy containers auto-restart and are fully managed via the Docker API. They are cleaned up when the plugin shuts down.

### Adding Cameras in UniFi Protect

1. Install and configure the plugin in Scrypted
2. Enable **Auto-assign unique IPs** and configure the IP range, network interface, subnet, and gateway
3. The plugin automatically creates proxy containers (visible via `docker ps | grep onvif`)
4. In UniFi Protect: **Settings > Cameras > Add Camera > ONVIF**
5. Enter each camera's assigned IP with port `8000`
6. Enter the ONVIF username/password you configured

### Unraid Notes

- If your Scrypted container uses `Custom: br0.2` (ipvlan), set the **Network interface** to `br0` instead ŌĆö this gives each proxy container a unique MAC
- The plugin creates a Docker network called `onvif_cameras` for the proxy containers
- Proxy containers are named `onvif-proxy-{deviceId}` and use the `unless-stopped` restart policy
- The `alpine/socat` image is pulled automatically on first use (requires internet access)

## ONVIF Services

The plugin implements the following ONVIF services per camera:

### Device Service (`/onvif/device_service`)
- `GetSystemDateAndTime` ŌĆö Returns current UTC time
- `GetDeviceInformation` ŌĆö Manufacturer, model, firmware, serial from Scrypted
- `GetCapabilities` ŌĆö Advertises Media, PTZ, Events based on device capabilities
- `GetServices` ŌĆö Lists all available service endpoints
- `GetScopes` ŌĆö ONVIF scopes (name, hardware, profile, type)
- `GetNetworkInterfaces` ŌĆö Network information with unique MAC per camera

### Media Service (`/onvif/media_service`)
- `GetProfiles` ŌĆö One profile per RTSP rebroadcast stream with video/audio configuration
- `GetStreamUri` ŌĆö Returns the RTSP rebroadcast URL for the requested profile
- `GetVideoSources`, `GetAudioSources` ŌĆö Source configurations
- `GetSnapshotUri` ŌĆö Snapshot endpoint (when available)

### PTZ Service (`/onvif/ptz_service`) *(when PTZ is available)*
- `ContinuousMove` / `Stop` ŌĆö Pan, tilt, zoom control
- `GetConfigurations` / `GetNodes` ŌĆö PTZ capability information
- `GotoPreset` / `SetPreset` / `GetPresets` ŌĆö Preset management

### Event Service (`/onvif/event_service`)
- `CreatePullPointSubscription` ŌĆö Subscribe to events
- `PullMessages` ŌĆö Poll for motion and object detection events
- `Renew` / `Unsubscribe` ŌĆö Subscription management

## How It Works

1. When Scrypted loads the plugin, it attaches as a mixin to every camera/doorbell
2. The mixin discovers all RTSP rebroadcast URLs from the Rebroadcast plugin
3. An HTTP server starts on a per-camera port, serving ONVIF SOAP responses
4. If auto-assign IPs is enabled, a Docker macvlan proxy container is created with a unique IP and MAC
5. WS-Discovery announces the camera on the local network via UDP multicast
6. Motion and object detection events from Scrypted are queued and served via PullPoint

## Troubleshooting

- **Camera not discovered** ŌĆö Ensure the ONVIF server is enabled in the camera mixin settings and check the plugin console for errors
- **Authentication errors** ŌĆö Verify username/password match in both plugin settings and the ONVIF client
- **No streams in profiles** ŌĆö Make sure the Rebroadcast plugin is installed and the camera has RTSP rebroadcast URLs configured
- **Port conflicts** ŌĆö If a port is in use, the plugin automatically assigns a random available port and saves it
- **UniFi sees one camera** ŌĆö Enable auto-assign IPs; without unique MACs, UniFi treats all cameras as the same device
- **Proxy containers not starting** ŌĆö Check that Docker socket is mounted and `alpine/socat` image is available (in air-gapped environments, pre-pull it: `docker pull alpine/socat`)
- **Enable Debug events** to see detailed motion/detection event forwarding in the console
