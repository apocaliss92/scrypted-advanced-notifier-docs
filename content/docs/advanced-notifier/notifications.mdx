# Notifications

The plugin delivers customized notifications based on rules. Each rule can be triggered by sensors, time ranges, security system status, and more.

## Supported Notifiers

| Notifier | Notes |
|----------|-------|
| **Zentik** | Full-featured app: iOS, watchOS, iPad, macOS. Push, history, custom actions. [Setup guide](/docs/zentik) |
| **Native Scrypted** | Scrypted iPhone app |
| **Ntfy** | [ntfy.sh](https://ntfy.sh) |
| **Home Assistant** | Push notifications |
| **Pushover** | Push notifications |
| **Telegram** | Telegram bot |

**Zentik** is recommended for the best experience: rich notifications, history, and actions. For other setups, you can combine notifiers (e.g. Pushover/Ntfy for storage, Home Assistant/NVR for rich notifications with actions).

## General Notifier Settings

When you extend a notifier with Advanced Notifier, each notifier device gets its own settings panel. Configure them under the notifier device page in Scrypted.

![Notifier general settings](/assets/Notifications/Notifier%20general%20settings.png)

| Setting | Description |
|---------|-------------|
| **Enabled** | Enable or disable this notifier |
| **Translations** | Use the plugin **Texts** section to customize notification messages |
| **AI descriptions** | Use configured AI to generate notification text from the image. For NVR: enable on both the notifier and each camera |
| **Notification key to edit** | Which payload key the plugin edits (body, title, subtitle, etc.). Use especially with **Scrypted NVR notifier** devices when texts are not displayed correctly — iOS devices may map title/subtitle/body differently across notifiers |
| **Enable on all notifications** | Testing setting |
| **Scheduler** | Restrict notifications to a time range |
| **Start time / End time** | Time window when notifications are sent (when Scheduler is enabled) |

## Notification Text Generation

Notification text is built from the **Texts** section of the plugin. These texts are used both for **rules** (detection, occupancy, timelapse, etc.) and as **translations for NVR notifier** messages. You can define templates with placeholders like `${room}`, `${time}`, `${classnameText}`, `${nvrLink}`.

### Plugin Texts (global)

Configure default texts in **Advanced Notifier** → **Texts**. These apply to all notifiers unless overridden.

![Texts translations](/assets/Notifications/Texts%20translations.png)

### Override on Notifier

Each extended notifier can override the plugin texts. Enable **Translations** in the notifier settings, then edit the **Texts** subgroup on that notifier.

![Notifier texts translations](/assets/Notifications/Notifier%20texts%20translations.png)

### Detection Classes

The **Detection classes** subgroup lets you customize labels for each detection type (Person, Animal, Vehicle, Motion, etc.). Useful for localization or custom wording.

![Classes translations](/assets/Notifications/Classes%20translations.png)

## Camera-Specific Notification Settings

Per-camera notification options are under **Advanced Notifier** → **Notifier** on each camera.

![Camera notifier settings](/assets/Notifications/Camera%20notifier%20settings.png)

| Setting | Description |
|---------|-------------|
| **Notifications enabled** | Enable or disable notifications for this camera |
| **AI descriptions** | Use AI to generate notification text (requires AI configured in plugin). For NVR: enable on both the notifier and each camera |
| **Scheduler** | Restrict notifications to a time range for this camera |
| **Start time / End time** | Time window when notifications are sent |
| **Default actions** | Actions shown on every notification. See [Actions](#actions) below. |

## Actions

Actions are buttons shown on push notifications (e.g. open door, snooze, view in app). They can be set per camera (**Default actions**) or per rule. **Support and behavior depend on the selected notifier**:

| Notifier | Rule-level actions | Camera default actions | Notes |
|----------|--------------------|-------------------------|-------|
| **Zentik** | Yes (Detection rules) | Yes | Full support. Use `sfsymbols:*` for icons on iOS |
| **Home Assistant** | Yes (Detection rules) | Yes | Leave `url` empty to use HA webhook — `action` becomes the script/automation ID |
| **NVR** (Scrypted) | No | Yes | Only camera default actions; rule-level actions not available |
| **Ntfy** | No | Yes | Limited; actions may not render on all clients |
| **Pushover** | No | No (default off) | Actions not added by default |
| **Telegram** | No | Yes | Depends on Telegram bot capabilities |

### Expected structure

Each action is a JSON object with the following fields:

| Field | Required | Description |
|-------|----------|-------------|
| `action` | Yes | Unique identifier. For Home Assistant with empty `url`, this is the script/automation ID |
| `title` | Yes | Label shown on the button |
| `url` | Yes | URL opened when tapped. Use `""` for Home Assistant — the plugin builds the webhook from `action` |
| `icon` | No | `sfsymbols:door` (iOS) or `mdi:webcam` (Material Design) |

### Examples

**Generic (URL-based):**
```json
{"action":"open_door","title":"Open door","icon":"sfsymbols:door","url":"https://home.example.com/script/open_door"}
```

**Home Assistant (empty url — plugin builds webhook from `action`):**
```json
{"action":"open_door","title":"Open door","icon":"mdi:door","url":""}
```

## AI Descriptions

AI descriptions use an LLM to generate notification text from the snapshot image. You must first configure an AI provider in [AI Setup](/docs/advanced-notifier/ai-setup).

AI can be used in two ways:

- **Rules** — Enable **AI descriptions** in the rule configuration (Rules → Detection / Occupancy / etc. → per-rule settings). No other setup required.
- **NVR notifications** — Enable **AI descriptions** on both the **notifier** and each **camera** (cameras must also have AI descriptions enabled). See the images above: [General Notifier Settings](#general-notifier-settings) and [Camera-Specific Notification Settings](#camera-specific-notification-settings).
