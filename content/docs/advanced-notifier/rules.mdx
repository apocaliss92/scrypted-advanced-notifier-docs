# Rules

Rules can be: **Detection**, **Occupancy**, **Audio**, **Timelapse**, **Recording**. Some properties are hidden until **Show more configurations** is activated.

## Common Properties

| Property | Description |
|----------|-------------|
| **Activation type** | When the rule is active: Always, Schedule (time range), OnActive (devices selector) |
| **Notifiers** | Notifiers to notify; Pushover priority, Home Assistant actions |
| **Open sensors** | Sensors that must be open to enable the rule |
| **Closed sensors** | Sensors that must be closed to enable the rule |
| **Alarm modes** | Alarm states that enable the rule (Security system in Rules → General) |
| **Notify with a clip** | (Detection, Occupancy) Generate short clip instead of snapshot. MP4 (HA) or GIF (HA, Pushover) |

### OnActive Activation

When activation type is **OnActive**, the rule is active only if the camera is in the **OnActive devices** selector (Plugin → Rules → General). The selector can be driven by MQTT via **Active entities topic** under General → MQTT. The message can contain device IDs, names, or Home Assistant entity IDs.
