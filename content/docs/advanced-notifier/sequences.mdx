# Sequences

In the **Sequences** section (plugin page) you define custom sequences. Each sequence is a list of steps executed in order. Attach sequences to rules at different moments.

## Step Types

| Step | Description |
|------|-------------|
| **Wait** | Pause for a number of seconds before the next step |
| **Script** | Run a Scrypted script (Program device). On-generated scripts receive `variables.payload` |
| **PTZ** | Move camera to a PTZ preset (requires PanTiltZoom device) |
| **Switch** | Turn a switch or light on or off (OnOff device) |
| **Lock** | Lock or unlock a lock device |
| **Entry** | Open or close an entry (garage door, gate, etc.) |

## Attachment Points

Attach sequences to rules in **Rules** → per-rule → **Sequences**:

| Moment | When it runs |
|--------|--------------|
| **On-activation** | When the rule becomes active (rules with activation type other than Always) |
| **On-deactivation** | When the rule becomes inactive |
| **On-trigger** | When the rule is triggered (e.g. detection matched, occupancy change) |
| **On-reset** | When the rule is reset (e.g. after trigger window ends) |
| **On-generated** | When all artifacts are ready (video clip, GIF, image). Detection, Occupancy, Timelapse |

## On-generated Payload

For **On-generated sequences**, scripts receive a payload via `variables.payload` in Scrypted scripts.

**Detection and Occupancy:**

- `rule` — rule object (name, notifiers, settings)
- `videoUrl` — URL of generated video clip (if MP4)
- `gifUrl` — URL of generated GIF (if GIF)
- `imageUrl` — URL of stored snapshot

**Timelapse:**

- `rule` — timelapse rule object
- `videoUrl` — URL of timelapse video
- `imageUrl` — URL of thumbnail
- `videoPath`, `imagePath` — filesystem paths (when available)

## Sequence Settings

Each sequence has:

- **Enabled** — Turn the sequence on or off
- **Minimum execution delay** — Minimum seconds between consecutive runs of the same sequence (prevents rapid re-triggering)
