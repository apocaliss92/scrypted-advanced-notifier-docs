# Detection Rules

Detection rules specify which object detections trigger a notification. Can be created for multiple cameras (plugin page) or per single camera.

## Setup

1. Add a new text in **Detection rules** selector and save
2. Set activation type
3. Set notifiers
4. Configure detection classes (and score threshold in Show more if needed)

## Options

| Setting | Description |
|---------|-------------|
| Detection classes | Which objects trigger (motion, animal, person, vehicle, etc.) |
| Whitelisted zones | Use only detections in these zones |
| Blacklisted zones | Ignore detections from these zones |
| Custom text | Override default texts (placeholders available in Texts tab) |
| Enable AI descriptions | AI generates description from image |
| CLIP Description | Semantic search to filter detections |
| AI filter | Send image to AI to confirm input prompt |
| Image post processing | MarkBoundaries (draw box), Crop (crop around object) |
| Score threshold | Minimum score to trigger (Show more) |
| Minimum notification delay | Debounce notifications (Show more) |
| Minimum MQTT publish delay | Debounce MQTT image updates (Show more) |
| Disable recording in seconds | Enable NVR recording for X seconds then disable (Show more) |

## Audio Classification

Detection rules can detect audio labels (crying, scream, speech, etc.) by adding the **Audio** label in detection classes. A new setting appears for which labels to consider.

**Classifier** (Audio analysis section of camera):

- **YAMNET** — YAMNet Audio Classification plugin (onboard analyser)
- **DISABLED** — No onboard classifier; external plugins (e.g. Frigate Bridge) can still forward events
