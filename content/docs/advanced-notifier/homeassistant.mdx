# Home Assistant

The Home Assistant connection is required to use **Home Assistant notifiers** for push notifications. The plugin also generates automations needed for **notification snoozing** (snooze actions on push notifications) and for the **onboard [alarm system](/docs/advanced-notifier/alarm-system)** (arming, disarming, triggering).

## HA Custom Integration (HACS)

If you use **WebSocket transport** (recommended), install the custom integration via HACS:

[![Open your Home Assistant instance and open a repository inside the Home Assistant Community Store.](https://my.home-assistant.io/badges/hacs_repository.svg)](https://my.home-assistant.io/redirect/hacs_repository/?owner=apocaliss92&repository=scrypted-advanced-notifier-homeassistant&category=integration)

Or manually:

1. In HACS, add `https://github.com/apocaliss92/scrypted-advanced-notifier-homeassistant` as a custom repository (category: Integration)
2. Install **Scrypted Advanced Notifier**
3. Restart Home Assistant

### Configure the Integration

After installing via HACS:

1. Go to **Settings → Integrations → Add Integration** and search for *Scrypted Advanced Notifier*
2. The config flow asks for two parameters:

![Config flow — Scrypted URL and HA Secret](/assets/Homeassistant/HA%20-%20add%20integration%201.png)

| Parameter | Where to find it |
|-----------|-----------------|
| **Scrypted URL** | The base URL of your Scrypted instance, e.g. `https://192.168.1.4:10443`. This must be reachable from Home Assistant. |
| **HA Secret** | In the Advanced Notifier plugin: **General → Homeassistant → HA Secret**. This is an auto-generated shared secret — copy it exactly. |

3. After submitting, the integration discovers all devices and entities from the plugin:

![Discovered devices after setup](/assets/Homeassistant/HA%20-%20add%20integration%202.png)

4. You can manage which cameras are imported via **Configure → Manage devices**:

![Manage devices — select cameras](/assets/Homeassistant/HA%20-%20add%20integration%203.png)

### WebSocket Transport Setup

1. Go to **General → Homeassistant** and set **Discovery layer** to **websocket**
2. Copy the **HA Secret** into the HA custom integration config flow (see above)
3. Set **HA Allowed Origins** to your Home Assistant local address (e.g. `http://homeassistant.local:8123`). This is a comma-separated list of HA base URLs allowed to connect to the plugin.

![WebSocket transport settings](/assets/Homeassistant/HA%20-%20transport.png)

:::info No MQTT broker needed
With WebSocket transport, you do **not** need an MQTT broker. The plugin pushes state updates to the HA custom integration via REST and receives commands back via REST. No admin token or persistent WebSocket connection is required.
:::

### HA Secret Management

The **HA Secret** is a shared secret used to authenticate communication between the plugin and the HA custom integration. It is auto-generated when the plugin starts for the first time.

#### Regenerating the Secret

If you need to regenerate the secret (e.g. for security reasons), you have two options:

1. **Automatic (recommended):** Click the **Regenerate HA Secret** button in the plugin settings (**General → Homeassistant**). The plugin will generate a new secret and push it to the HA integration automatically. If HA is reachable, both sides are updated — no manual action required.

2. **Manual:** If the automatic push fails (e.g. HA is unreachable, or the secrets are already out of sync), you need to update the secret on HA manually:
   - Copy the new secret from the plugin settings (**General → Homeassistant → HA Secret**)
   - In HA, go to **Settings → Integrations → Scrypted Advanced Notifier → Configure** and select **Update HA Secret**:

   ![Configure menu — select Update HA Secret](/assets/Homeassistant/HA%20-%20update%20secret%201.png)

   - Paste the new secret and confirm:

   ![Paste the new HA Secret](/assets/Homeassistant/HA%20-%20update%20secret%202.png)

:::warning Secret mismatch
If the secret on the plugin does not match the one on HA, you will see `401 Unauthorized` errors in the plugin logs. The plugin will log a warning with instructions to fix the mismatch. Use one of the two methods above to re-sync the secret.
:::

For the list of discovered entities and shared settings, see [Discovery](/docs/advanced-notifier/discovery).

## Connection Steps

### Step 1: Enable Home Assistant

Go to **General → General** and enable **Homeassistent enabled**.

![Enable Home Assistant in General tab](/assets/Homeassistant/HA%20-%20enable.png)

### Step 2: Configure Credentials

Go to **General → Homeassistant**:

- **If the Home Assistant plugin is already configured in Scrypted** — check **Use HA plugin credentials** to reuse the URL and token from the Scrypted Home Assistant plugin.

![Use Home Assistant plugin credentials](/assets/Homeassistant/HA%20-%20auto%20login.png)

- **Otherwise** — uncheck **Use HA plugin credentials** and set **Protocol** (http/https), **Address** (e.g. `192.168.1.1:8123`), and **HA Personal access token** manually. Create a Long-Lived Access Token in Home Assistant under **Profile → Security → Long-Lived Access Tokens**.

![Manual Home Assistant credentials](/assets/Homeassistant/HA%20-%20manual%20login.png)

### Step 3: Scrypted Token (for proxy app)

The **Scrypted token** in **General → Homeassistant** is used only to allow opening the camera stream through the Home Assistant proxy app. This token is found on the Home Assistant entity created by the Scrypted integration (e.g. `sensor.scrypted_token_{ip}`).

![Scrypted token step 1](/assets/Homeassistant/HA%20-%20token%20step%201.png)

![Scrypted token step 2](/assets/Homeassistant/HA%20-%20token%20step%202.png)

![Scrypted token step 3](/assets/Homeassistant/HA%20-%20token%20step%203.png)

![Scrypted token step 4](/assets/Homeassistant/HA%20-%20token%20step%204.png)

### Step 4: Generate Automations

When Home Assistant is enabled, the plugin automatically generates two automations in Home Assistant:

1. **Snooze automation** (`scrypted_advanced_notifier_snooze_action`) — Handles snooze actions from push notifications (e.g. "Snooze 15 min").
2. **Alarm automation** (`scrypted_advanced_security_system_alarm_action`) — Handles alarm panel commands (arm, disarm, trigger) for the onboard alarm system.

These automations are created at plugin startup when **Homeassistent enabled** is on. If you enable Home Assistant after initial setup, restart the plugin or toggle the setting to trigger generation.
